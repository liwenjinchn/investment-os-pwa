# 投资 OS PWA

移动端优先、本地优先的投资决策 PWA。当前主流程是“事件决策”：把一条消息转成证据账本、情景树、纪律动作和到期复盘，而不是让新闻直接触发交易。

- 数据长期保存在浏览器 IndexedDB
- API key 只保存在本地浏览器
- 支持持仓、事件研判、证据分级、概率情景、组合规则、决策日志、周复盘、OCR 候选确认、备份导入导出
- 事件数据与旧版数据均保存在浏览器 IndexedDB；数据库会从 v1 自动升级到 v2
- 情景概率不等于 100%、缺少负面情景、一手证据或证伪条件时，确认动作会被阻止
- AI 只作为可选的反方审查，不是完成事件决策的前置条件
- 导出备份包含事件研判，但会自动清空 API Key
- GitHub Pages 静态部署

## Event decision flow

1. 定义事件、时间窗和受影响的原始论点。
2. 将证据标成事实 / 假设 / 判断，并按一手文件、公司口径、独立来源、市场信号、未核实分级。
3. 建立至少两个情景，概率合计 100%，保留明确下行情景与可观察信号。
4. 系统计算组合期望影响，并让现金、单仓与主题暴露规则拥有否决权。
5. 用户确认最终动作、理由、下一触发器与复盘日期；确认后写入监控和决策日志。
6. 到期追加实际结果和教训，不覆盖原研判。

完整产品契约与验收记录见 [`docs/event-decision-v1.md`](docs/event-decision-v1.md)。
